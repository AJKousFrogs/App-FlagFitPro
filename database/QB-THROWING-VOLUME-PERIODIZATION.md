## Critical Fix Applied

**Problem Identified**: Original seed data had throwing volume increasing from 100 to 320 throws in just 4 weeks (December). This is **dangerous and unrealistic**.

**Solution**: Gradual progression over **5 months** (November → April) with weekly increases of 10-20 throws max.

## Key Principles Applied

### ✅ 1. Gradual Build Over 5 Months

- **Total build time**: 22 weeks (November → April Week 2)
- **Total increase**: 50 → 320 throws = +270 throws
- **Average weekly increase**: ~12 throws/week
- **Maximum weekly jump**: 20 throws (only in Feb-March when base is established)

### ✅ 2. No Sudden Jumps in January

- **January Week 1**: 120 throws
- **January Week 2**: 130 throws (+10, NOT +50!)
- **January Week 3**: 140 throws (+10)
- **January Week 4**: 155 throws (+15)

### ✅ 3. Peak Volume by Season Start

- **April Week 2**: 320 throws achieved
- **Why Week 2?**: Week 1 includes small taper (315) before full season volume

### ✅ 4. MAINTAIN Through Entire Season

- **April-June (12 weeks)**: CONSTANT 320 throws/week
- **No increases during season**: Body already at peak capacity
- **Focus**: Recovery, game performance, ACWR monitoring

### ✅ 5. RPE Monitoring Critical

- **Why?**: Small weekly increases (10-20 throws) may seem safe, but cumulative fatigue builds
- **Track RPE every session**: If RPE stays high (8-9+) for multiple weeks, reduce volume
- **ACWR Sweet Spot**: Keep ratio between 0.8-1.3 to minimize injury risk

## Player Instructions

### How to Track Throwing Volume

1. **Log Every Throwing Session**
   - Count ALL throws (warm-up, drills, routes, game simulation)
   - Record in `workout_logs` with RPE
   - Add to `position_specific_metrics` table

2. **Weekly Totals**
   - Sum all throwing sessions for the week
   - Compare to target (e.g., Week 10 = 130 throws)
   - Stay within ±10% of target

3. **Listen to Your Arm**
   - If RPE > 8 for multiple sessions → Tell coach
   - Arm soreness > 48 hours → Rest day
   - Sharp pain → STOP immediately

4. **Trust the Process**
   - 50 throws in November feels easy → GOOD
   - 320 throws in April feels manageable → PERFECT
   - Don't rush the build

## Database Queries for Monitoring

### Check Current Week's Target Volume

```sql
SELECT
  tw.week_number,
  tw.focus,
  (tw.volume_multiplier * 100) AS target_throws,
  tw.start_date,
  tw.end_date
FROM training_weeks tw
WHERE tw.start_date <= CURRENT_DATE
  AND tw.end_date >= CURRENT_DATE
  AND tw.phase_id IN (
    SELECT id FROM training_phases WHERE program_id = '11111111-1111-1111-1111-111111111111'
  );
```

### Check Player's Actual Throwing Volume This Week

```sql
SELECT
  SUM(psm.metric_value) AS actual_throws_this_week
FROM position_specific_metrics psm
WHERE psm.player_id = 'player-uuid'
  AND psm.metric_name = 'Throwing Volume'
  AND psm.date >= DATE_TRUNC('week', CURRENT_DATE)
  AND psm.date < DATE_TRUNC('week', CURRENT_DATE) + INTERVAL '7 days';
```

### Compare Target vs. Actual

```sql
WITH target AS (
  SELECT (tw.volume_multiplier * 100) AS target_throws
  FROM training_weeks tw
  WHERE tw.start_date <= CURRENT_DATE AND tw.end_date >= CURRENT_DATE
),
actual AS (
  SELECT SUM(metric_value) AS actual_throws
  FROM position_specific_metrics
  WHERE player_id = 'player-uuid'
    AND metric_name = 'Throwing Volume'
    AND date >= DATE_TRUNC('week', CURRENT_DATE)
)
SELECT
  target.target_throws,
  actual.actual_throws,
  (actual.actual_throws - target.target_throws) AS difference,
  CASE
    WHEN ABS(actual.actual_throws - target.target_throws) <= (target.target_throws * 0.1)
      THEN '✅ On Track'
    WHEN actual.actual_throws > target.target_throws
      THEN '⚠️ Over Target'
    ELSE '⚠️ Under Target'
  END AS status
FROM target, actual;
```
