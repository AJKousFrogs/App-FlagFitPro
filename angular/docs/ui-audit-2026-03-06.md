# UI Audit - 2026-03-06

## Scope

This audit covers the Angular frontend in `angular/` with emphasis on the design-system contract, shared SCSS architecture, auth flows, and route-level UI duplication cleanup.

This report reflects the repo after the token, layout, forms, feedback, empty-state, motion, and header cleanup passes were implemented and verified.

## Current Design-System Structure

### Canonical entrypoints
- Angular styles entry: `angular/angular.json -> projects.flagfit-pro.architect.build.options.styles`
- Global stylesheet entry: `angular/src/styles.scss`
- Canonical tokens: `angular/src/scss/tokens/design-system-tokens.scss`
- Design-system contract: `angular/src/scss/README.design-system.md`

### Effective import graph
`angular/src/styles.scss` currently composes the stack in this order:
1. `styles/design-system/index`
2. `styles/design-system/elite-command-center.tokens`
3. `styles/design-system/elite-phase2-layout`
4. `styles/design-system/elite-phase2-auth`
5. `scss/utilities`
6. `assets/fonts/poppins`
7. `scss/foundations`
8. `scss/components/index`
9. `scss/pages/index`
10. `assets/styles/overrides`
11. vendor `primeicons.css`

### Canonical ownership
- Tokens: `src/scss/tokens/**`
- Utilities and foundations: `src/scss/utilities/**`, `src/scss/foundations/**`
- Reusable primitives/components: `src/scss/components/**`, `src/app/shared/components/**`
- Page utilities and route enhancers: `src/styles/design-system/**`, `src/scss/pages/**`
- Exceptions/PrimeNG overrides: `src/assets/styles/overrides/**`

### Current architecture assessment
The repo now behaves like a real token-first system:
- tokens
- utilities/foundations
- reusable primitives
- pages
- overrides

The style-root split is now formalized:
- canonical modules live in `src/scss/design-system/**`
- `src/styles/design-system/**` is compatibility-only

That removes active ownership ambiguity while keeping backward compatibility for old imports.

## What Was Broken In The UI

### Before this cleanup
- auth screens had inconsistent hierarchy, detached error feedback, and duplicated form behavior
- cards, alerts, dialogs, banners, and toasts used multiple spacing/radius/shadow patterns
- page shells used ad-hoc spacing instead of shared layout utilities
- many route-level components reimplemented headers, empty states, alerts, and summary tiles
- dark mode depended on older aliases rather than a clean semantic surface/text layer

### Current state
No critical broken UI remains in the audited high-traffic routes.

Routes that now read as production-ready and aligned:
- `auth/login`
- `auth/register`
- `auth/reset-password`
- `player-dashboard`
- `today`
- `training`
- `roster`
- `settings`

What remains is not broken UI. It is normal future product/UI iteration and a small amount of architectural cleanup around style ownership.

## Duplicate SCSS Patterns And Consolidation Results

### Consolidated
- semantic tokens and component tokens in `src/scss/tokens/design-system-tokens.scss`
- native + PrimeNG form behavior in `src/scss/components/primitives/_forms.scss`
- cards in `src/scss/components/primitives/_cards.scss`
- dialogs in `src/scss/components/primitives/_dialogs.scss`
- alerts/toasts/banners in `src/scss/components/notifications.scss`
- motion in `src/scss/utilities/_motion.scss`
- metric/summary primitives in `src/scss/components/primitives/_metrics.scss`
- shared page shell/grid/stack utilities in `src/styles/design-system/utilities.scss`
- shared card header primitive in `src/app/shared/components/card-header/**`

### Removed or reduced
- route-local `p-message` usage
- duplicated alert/banner shells
- duplicated modal wrapper usage
- duplicated close-button/backdrop logic
- duplicated `empty-state-card` wrappers
- duplicated `loading-placeholder` shells
- duplicated generic `summary-item` / `insight-item` / `stat-summary-item` patterns
- duplicated generic `card-header` / `chart-header` / `benchmark-header` patterns
- large dead compatibility layer in `src/scss/components/standardized-components.scss`

### Remaining consolidation candidates
- eventual deletion of `src/styles/design-system/**` compatibility shims once all legacy imports are retired
- selective cleanup of untouched vendor/stylelint noise
- documentation tightening around when to use `app-card-header` vs the internal shared `card.component`

## Implemented Outcomes

### Tokens upgrade
- semantic surface, text, border, status, focus, layout, and component tokens are defined
- spacing scale is standardized around `4/8/12/16/24/32/48/64`
- component tokens cover button/input/card/dialog/toast geometry
- dark and light mode use the same semantic layer

### Layout system upgrade
- shared page shell, section card, cluster, grid, and stack utilities are in place
- page rhythm no longer depends on ad-hoc margins in the upgraded routes

### Forms system upgrade
- neutral control surfaces
- 48px default input height
- standardized radii
- focus ring tokens
- inline field/form errors
- standardized helper text, labels, password toggle, and checkbox layout

### Reusable component styling upgrade
- buttons standardized across primary/secondary/tertiary/icon-button paths
- cards/panels standardized through shared tokens and primitives
- alerts/toasts/modals/banners consolidated to canonical implementations
- empty/loading/page-error states standardized
- header primitives standardized

### Broken UI fixes
- login/auth hierarchy is modern, centered, responsive, and inline-feedback driven
- dashboard and today routes now match the same system
- high-traffic and previously duplicated route families are aligned to the same primitives
