# V2.3 — Bug Sweep

**Status: one real bug found and fixed in new V2 code; repo-wide sweep otherwise clean.** This phase ran ESLint, `tsc --noEmit`, `ng build` (AOT), the full Angular + backend test suites, and the repo's own guard scripts (consent-violation check, schema-drift check, knowledge-base validator/secret scanner) across the whole codebase — not just this session's changes.

## Fixed

### Timezone bug in `tournament-plan.service.ts` (V2.0 gap engine) — real, would have shipped broken

`toMinutes`/`fromMinutes` round-tripped kickoff times through `new Date(`${date}T${time}`)` — which parses a date+time string with no timezone as the **host's local time** — then read the result back with `.getUTCHours()`/`.getUTCMinutes()`. Those two are only the same clock when the host happens to run in UTC. The test sandbox does (`Intl.DateTimeFormat().resolvedOptions().timeZone === "UTC"`), which is exactly why all 23 original spec tests passed despite the bug — a browser anywhere else (a Ljubljana Frogs athlete's phone at UTC+1/+2) would have every generated timeline time silently shifted by the local UTC offset. Since `event_games.kickoff_time` is deliberately venue-local wall-clock with no timezone (by design — the gap engine only needs local deltas, never a real instant), this was the wrong tool from the start.

**Fix:** rewrote both helpers as pure integer day/minute arithmetic with zero `Date`-object involvement (`Date.UTC(y, m-1, d)` for a timezone-safe day index from explicit numeric components, plain `h*60+m` for time-of-day). No behavior change on a UTC host; correct everywhere else.

**Caught by:** manually re-deriving the epoch-based math while re-reading the V2.0 diff for this sweep, not by any automated tool — `tsc`/ESLint/the original tests all passed with the bug in place because the bug only manifests when `TZ !== UTC`. This is exactly why a regression test that changes `TZ` was added (see below) — without it, this class of bug is invisible to CI unless CI happens to run in a non-UTC zone.

**Regression coverage added:** two new spec tests in `tournament-plan.service.spec.ts` (25 total now, up from 23):

1. Rebuilds the Capital Bowl plan under `TZ=Pacific/Kiritimati` (UTC+14) and asserts kickoff times still render as `11:00`/`12:30`/`15:30`/`17:00` — would have failed against the pre-fix code (traced through by hand: 11:00 parsed as UTC+14 local → 21:00 UTC the prior day → `.getUTCHours()` reads back `21`, not `11`).
2. A two-day tournament boundary (day-1 game at 20:00, day-2 game at 09:00) — confirms the day-index arithmetic correctly spans midnight (740-minute gap, not a wrapped/negative one).

### Nonexistent CSS class in V2.0's gameday kickoff editor

`gameday.component.html`'s bulk-error message used `class="text-danger"`, which isn't defined anywhere in the stylesheet (silently unstyled — not a functional break, but wrong). Found and fixed while building V2.2 (same pattern was about to be copy-pasted into `qb-arm-care-card.component.ts`). Both now use the actual convention: `<p class="note" style="color:var(--danger)">`, matched from `wellness.component.html`/`onboarding.component.html`.
